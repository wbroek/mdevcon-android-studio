Gradle

New build system for Android

All new build system

  • Java-based build management and automation tool
  • Made by Gradleware
  • Not only Java, Groovy and Scala but also C++ and Objective-C
  • Simple readable build files with Groovy DSL
  • Build output everywhere the same
  • Plugin based
  • Android Gradle plugin entirely by Google
  • Will totally replace Ant
  • You will edit it

Import to Android Studio

What the hell!? Error?

Imported Gradle fix

Open the build.gradle and replace

            dependencies {
                classpath 'com.android.tools.build:gradle:0.5.+'
            }
          
            dependencies {
                classpath 'com.android.tools.build:gradle:0.9.+'
            }

When done use

NOTE: 0.9.+ is current version of used Gradle will change in future

Imported Gradle fix Android target

Open the build.gradle and replace

            android {
                compileSdkVersion 17
                buildToolsVersion "17.0.0"
            }
          
            android {
                compileSdkVersion 19
                buildToolsVersion "19.0.1"
            }

When done use

Import caveats

  • Could not find element “Application”
  • Application tag must be in manifest of all imported projects
  • Manifest Merging Failed
  • min and target SDK must be same for all projects
  • Wrong plugin added for library
  • change apply plugin: "android" to apply plugin: "android-library"
